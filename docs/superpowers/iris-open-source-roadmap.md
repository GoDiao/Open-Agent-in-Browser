# Iris 开源发布路线图

> 项目：Open-Agent-in-Browser (Iris)
> 日期：2026-06-04
> 目标：将 Iris 打造成可对外开源的 Chrome AI 助手扩展

---

## 📊 当前状态

| 维度 | 状态 | 评级 |
|------|------|------|
| 代码质量 | 架构清晰、67 工具完整、Memory+SOUL 系统已实现 | ✅ 优秀 |
| 测试覆盖 | 零测试、零 CI | ❌ 缺失 |
| 合规文档 | 无隐私政策、无用户协议、无 LICENSE | ❌ 缺失 |
| 安全机制 | API Key 明文存储、无操作确认 | ❌ 缺失 |
| 运营基础设施 | 无埋点、无灰度、无回滚 | ❌ 缺失 |
| UI 体验 | 视觉优秀但加载/错误状态缺失 | ⚠️ 部分 |

---

## 🎯 发布目标

**版本：1.0.0 (Alpha)**

具备条件：
- 核心功能可用（Agent + 记忆 + SOUL + 定时任务）
- 文档齐全（README + 架构 + 安全性说明）
- 合规准备（隐私政策 + 用户协议）
- 质量门禁（基础测试 + CI）

## 📅 路线图

### Phase 1: 合规与安全基础 🔴 阻断项
**预估：3-5 人天**

| 序号 | 任务 | 描述 | 优先级 |
|------|------|------|--------|
| 1.1 | 添加 LICENSE 文件 | 选择 MIT 或 Apache 2.0 许可证 | P0 |
| 1.2 | 起草隐私政策 | 说明数据收集范围、存储方式、第三方分享、用户权利 | P0 |
| 1.3 | 起草用户协议 | 服务条款、使用限制、责任声明 | P0 |
| 1.4 | 添加用户同意机制 | memory-review 自动提取需用户 opt-in/opt-out | P0 |
| 1.5 | 选中文本捕获说明 | 告知用户、提供禁用选项 | P1 |
| 1.6 | API Key 安全提示 | 在 Settings 中增加安全提示文案 | P1 |

### Phase 2: 核心功能修复 🔴 阻断项
**预估：2-3 人天**

| 序号 | 任务 | 描述 | 优先级 |
|------|------|------|--------|
| 2.1 | 修复 Network 工具 | 实现页面网络请求注入 + Fetch 事件监听 | P0 |
| 2.2 | 修复 get_network_requests | 确保能返回实际网络请求 | P0 |
| 2.3 | CDP 并发锁 | 为 attach 添加 per-tabId pending 队列 | P1 |
| 2.4 | Service Worker 保活 | AgentLoop 运行期间维持 SW 存活 | P1 |

### Phase 3: 文档补全 📚
**预估：2-3 人天**

| 序号 | 任务 | 描述 | 优先级 |
|------|------|------|--------|
| 3.1 | 完善 README.md | 项目概述、功能截图、快速开始、架构概览 | P0 |
| 3.2 | 安全性说明文档 | 明确 chrome.debugger 权限用途、数据存储位置 | P0 |
| 3.3 | 配置指南 | 支持的 LLM Provider、API Key 配置方式 | P1 |
| 3.4 | 权限说明文档 | 逐条解释 manifest.json 中权限的实际用途 | P1 |
| 3.5 | 添加 CONTRIBUTING.md | 贡献流程、编码规范、新工具添加指南 | P2 |

### Phase 4: UI/UX 完善 🎨
**预估：4-6 人天**

| 序号 | 任务 | 描述 | 优先级 |
|------|------|------|--------|
| 4.1 | Skeleton 加载组件 | 所有异步页面添加加载态 | P0 |
| 4.2 | 统一错误反馈 Toast | 所有异步写入操作添加错误反馈 | P0 |
| 4.3 | Tool Call Error 状态串联 | 确保工具执行失败时 UI 正确显示 | P1 |
| 4.4 | 触屏适配修复 | Quick Links 删除按钮改为常显 | P2 |
| 4.5 | Onboarding 无障碍修复 | Back 按钮 hidden 属性修复 | P2 |

### Phase 5: 质量门禁 🧪
**预估：3-4 人天**

| 序号 | 任务 | 描述 | 优先级 |
|------|------|------|--------|
| 5.1 | 引入 vitest | 配置测试框架 | P0 |
| 5.2 | 单元测试 | 为 memory.ts、soul.ts、framework.ts 编写测试 | P1 |
| 5.3 | GitHub Actions CI | 配置 lint + typecheck + build 自动化 | P1 |
| 5.4 | 类型测试 | 使用 z.infer 验证 tool handler 类型 | P2 |

### Phase 6: 运营基础设施 📊
**预估：2-3 人天**

| 序号 | 任务 | 描述 | 优先级 |
|------|------|------|--------|
| 6.1 | 匿名遥测埋点 | 可选 opt-in，打印关键事件 | P1 |
| 6.2 | 版本号启用 semver | 从 0.1.0 开始 | P1 |
| 6.3 | 数据导出功能 | 支持 JSON 导出记忆、配置、对话 | P1 |
| 6.4 | 数据导入功能 | 支持 JSON 导入恢复数据 | P2 |
| 6.5 | schemaVersion 字段 | 为 storage 添加版本控制 | P2 |

### Phase 7: 安全增强 🛡️
**预估：1-2 人天**

| 序号 | 任务 | 描述 | 优先级 |
|------|------|------|--------|
| 7.1 | 危险操作确认 | 删除历史、书签、执行脚本前二次确认 | P0 |
| 7.2 | 操作审计日志 | 记录关键操作到本地日志 | P1 |
| 7.3 | SSE 解析加固 | 处理 event: 行，忽略非 JSON 行 | P2 |
